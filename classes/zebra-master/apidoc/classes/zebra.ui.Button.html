The method is designed to customize mapping between the component state and a string view id. It can be handy for classes that extend the component to have own view IDs set.

Parameters:
  • b <Boolean>

    true if the component can gain focus

setFireParams( b, time )

Set the mode the button has to fire events. Button can fire event after it has been unpressed or immediately when it has been pressed. Also button can start firing events periodically when it has been pressed and holden in the pressed state.

Parameters:
  • b <Boolean>

    true if the button has to fire event by pressed event

  • time <Integer>

    the period of time the button has to repeat firing events if it has been pressed and holden in pressed state. -1 means event doesn't have repeated

setFocusAnchorComponent( c )

Set the specified children component to be used as focus marker view anchor component. Anchor component is a component over that the focus marker view is painted.

Parameters:
  • c <zebra.ui.Panel>

    an anchor component

setFocusMarkerView( c )

Set the view that has to be rendered as focus marker when the component gains focus.

Parameters:
  • c <String | zenra.ui.view | Fucntion>

    a view. The view can be a color or border string code or view or an implementation of zebra.ui.View "paint(g,x,y,w,h,t)" method.
